## Scope Rehearsed

- Analytics dashboard, session history, session detail, and PDF export were checked.
- Optional Cloud STT path was attempted.
- Native Browser STT was tested as the available alternate engine with real speech.

## Tester State Rule

- A session should be treated as recording only after the tester deliberately clicks the microphone / `Start Recording` control and that action succeeds.
- States such as `Mic ready`, `Ready to record`, model cached/ready, runtime ready, or navigation to `/session` are only preconditions. They are not recording evidence and should not create or imply an active session.
- If the app shows `Recording active` after a failed or ambiguous microphone/start action, treat that as a recording-state truth bug, not as proof that a valid tester session began.
