Architectural Clarifications:

1. ProtonMail vs Resend: ProtonMail is an incredibly secure inbox for humans to read/write email. However, apps need to send thousands of automated, programmatic emails instantly (like Magic Login Links, Password Resets, or "Welcome!" emails) via API. ProtonMail does not offer this developer API. Resend is the industry standard API for React apps to send transactional emails. It is completely free for up to 3,000 emails/month.

2. WebGPU & Vercel CDN: WebGPU runs the AI locally on the user's graphics card (zero cost to us). However, the user's browser must first download the 40MB "Whisper Voice Model" file to their computer before the graphics card can use it. We store this 40MB file on Vercel's Edge CDN. Vercel automatically delivers this file to the user for free (up to 1,000 GB of bandwidth a month). This keeps our Supabase costs at $0 for model delivery.

The "Gracious Sunset" Strategy (Soft Limits UX)

A hard error when a user wants to practice is a terrible experience. When users approach or hit their limits, we must use positive psychology to frame it as an achievement, while gently prompting the up-sell.

  • The 5-Minute In-Session Warning: If a user is 5 minutes away from hitting their Daily Target/Limit (or the 60m/120m session max), the recording bar smoothly turns yellow. Text updates to: "Great session! 5 minutes remaining for today's practice limit. Time to wrap up your thoughts."
  • Daily Limit Reached Modal: When the Daily 1h (Free) or 2h (Pro) limit is reached, we show a celebratory modal:
    "You've crushed your practice goals for today! 👏 Your brain needs time to absorb. Come back tomorrow for your next hour, or [Upgrade to Pro] to unlock double the daily bandwidth."
  • Monthly Limit Reached Modal:
    "You are in the top 1% of SpeakSharp users! 🏆 You've maximized your monthly 25-hour allowance. [Upgrade to Pro] for 50 full hours, or we'll see you on the 1st of next month!"
  • PDF Export Limit Reached (Free): When a Free user tries to click the PDF export for the 2nd time in a month, the button locks with a padlock icon:
    "You've used your 1 free teaser export this month! [Upgrade to Pro] for unlimited, un-watermarked exports."

App Misuse & Edge Case Mitigations (The 80/20 Pareto Analysis)

To ensure a rock-solid platform, we must anticipate how users will intentionally or accidentally break the app. Here are the top 20% of edge cases that would cause 80% of our infrastructure or support headaches, and our engineered solutions to stop them.

User Misuse / Edge Case The Risk Engineered Mitigation (Solution)
The "Overnight Sleeper" (Infinite Recording) Leaving the mic on overnight fills the browser Heap Memory until Chrome hard-crashes (OOM), deleting all their practice data and enraging the user. [IN PROGRESS] Graceful Limits: Hard bounding sessions to 60m/120m with a 5-minute UI warning before auto-saving and terminating the recording.
The "Dead Silence" Abuser A user walks away for an hour, recording total silence. This burns their daily limit on nothing, or wastes Cloud API processing time. [NEEDED] Voice Activity Detection (VAD) Pause: If no speech data is detected for 5 continuous minutes, auto-pause the session to protect the user's limit and our Cloud costs.
The "Multi-Tab Limit Bypass" A Free user opens 5 tabs simultaneously, recording in all of them to bypass the 1 hour/day limit before the database synchronizes. [NEEDED] Cross-Tab Mutex Lock: Use `localStorage` event listeners or a Supabase real-time presence lock to enforce that only *one* active recording session can exist per account globally.
The "Client-Side Hacker" A user modifies the React state or LocalStorage to grant themselves "Pro" status, accessing WebGPU. [NEEDED] Model URL Auth Proxy: Currently, WebGPU files are hosted on public CDNs. We must move the 60MB WASM/WebGPU models into a secure Supabase Storage Bucket. The client must request a signed URL. Supabase RLS policies will literally return `403 Forbidden` if the session's JWT `subscription_status` is not 'pro', utterly defeating client-side tampering.
The "War & Peace" Transcript A user successfully dictates 150 minutes of dense text, and the payload exceeds Gemini's context window or the 10-second Vercel serverless Function execution cutoff. [IN PROGRESS] The 50% Margin Of Safety: This 150-minute scenario will never happen *because* our enforced 120-minute Pro limit prevents it. But how do we know 120-minutes is safe? By running the Extreme Duration Soak test to find the absolute maximum limit, and cutting it in half.

Testing Methodology: The 50% Margin of Safety

To explicitly answer "How do we know the app won't crash when a user pays us?" we run a specialized automated test to find the absolute point of technical failure, and then set our marketing limits to exactly 50% of that failure point.

Step 1: Finding the Payload/Context Limit

We generate a massive chunk of dummy text simulating 4 straight hours of fast talking (~36,000 words). We feed this raw string into our `get-ai-suggestions` Edge Function. We systematically shrink the payload size until it successfully passes through the Gemini API without hitting token context limits, and successfully returns to the client without hitting Vercel's 10-second serverless timeout. If the maximum safe limit is 240 minutes of text, our Margin of Safety max allowance is 120 minutes.

Step 2: Finding the Browser RAM Limit

We use a headless Playwright Chrome instance to initialize the WebGPU pipeline. We pipe continuous audio into it and measure the Chrome Memory Heap every 10 minutes. When Chrome throws an `Out of Memory (OOM)` exception and crashes the tab, we note the timestamp. If the browser safely survived 4 hours before crashing, our 50% Margin of Safety limit is 120 minutes.

Conclusion: The 60m (Free) and 120m (Pro) constraints currently proposed in the PRD are hypothesized targets that must be validated by these two exact tests before launch.

Inversion Mental Model: How to Ruin SpeakSharp

"What would frustrate a customer or make them use the product once and leave?" Based on broad market research of existing STT complaints (Reddit, Google Forums) in 2024/2025.

❌ Guaranteed Ways to Frustrate the User:

  • The "Wait, what?" Model Load: Showing a blank screen or a tiny spinner for 30 seconds while the 40MB WebGPU Whisper model downloads. The user thinks the app is broken and closes the tab.
  • The "Opposite Day" Transcription: When the app changes context wildly (e.g., transcribing "do" as "don't"). Standard built-in device dictation does this constantly.
  • Zero Context Formatting: Dumping a massive block of unpunctuated text with no paragraphs or structure.
  • Privacy Paranoia: Sending highly confidential practice audio (medical/legal) to a random cloud server without explicit guarantees.
  • Browser Memory Crash: Letting them record for 4 hours straight on a 2GB RAM Macbook Air, causing the browser tab to crash and burning the transcript securely to the ground.

✅ Existing & New Tasks to Prevent This (Added to Roadmap):

  • [NEW] - Delightful Loading States: We must build beautiful, engaging skeleton loaders or "Model unpacking..." progress bars so the 20-second WebGPU download feels like a premium setup sequence.
  • [EXISTS] - Defeating "Opposite Day" (Triple-Engine): Native built-in dictation does word-by-word guessing, causing context errors. Whisper (Private STT) uses a Transformer architecture that looks at the entire sentence before guessing a word, making it highly context-aware. If local still struggles, Pro users can failover to Cloud.
  • [NEW] - "Vault Mode" Marketing UI: Add a visual "Vault Active" UI lock icon to prove NO network requests are being made during WebGPU processing. Attacks the #1 privacy fear head-on.
  • [NEW] - Graceful Session Wrap-up (Pending Test): Instead of a hard crash, users get a graceful 5-minute warning to wrap up before the session safely ends. *Note: The exact length of these bounds (60m vs 120m) requires an Extreme Duration Soak Test to verify 1) Browser RAM limits, and 2) If Gemini can successfully format a massive 2-hour text block without hitting token limits or API timeouts. Whatever the technical failure point is, we will apply a strict 50% Margin of Safety to our marketed limits.
  • [EXISTS] - Instant Polish (Gemini): We don't just dump text. The AI Coach automatically formats, structures, and highlights key areas. It can also spot logical contradictions caused by STT hallucinations.
